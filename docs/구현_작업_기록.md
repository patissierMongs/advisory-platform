## 1. 개요

### 1.1 배경 / 출발점
- **입력물 2종**: ① DC(Design Component) 프로토타입 `보안권고문 처리 시스템.dc.html` — 디자인·UX·상태머신은 훌륭하나 **백엔드 0의 목업**. ② **백엔드 구현 명세서**(10개 테이블·REST·파이프라인·게이트 정의).
- **판단**: 프로토타입의 `DCLogic`(매칭·그룹핑·메시지 생성)이 사실상 비즈니스 로직을 이미 구현 → 백엔드는 "하드코딩 배열을 DB로 치환 + 순수함수 포팅"이 핵심.

### 1.2 목표
1. 명세서대로 **동작하는 백엔드** 구현(국정원 권고문 PDF → CVE 추출 → 로컬 DB 조회 → 자산 매칭 → 부서 발송).
2. 기존 프론트엔드를 **UI는 그대로 둔 채 실 API에 연동**.
3. 후속 업무 요구(발송 후 루프·보고서·SLA·엑셀 견고화·오프라인) 완성.

### 1.3 결과
폐쇄망(오프라인) 운영 가능한 전체 시스템. `start.bat` 한 번으로 백엔드+UI 기동, 시드 데이터로 즉시 데모.

## 3. 구현 범위

### 3.1 명세 기반 핵심 (§1~5)
- **데이터 모델 13테이블**: department, asset, advisory, cve, advisory_cve, match, notification, cve_feed_import, asset_import(+mapping), audit_log, exclusion_rule, app_user. 이식성 위해 enum=VARCHAR+CHECK, JSONB=JSON.
- **파이프라인 4단계**: ① PDF 업로드+SHA-256 중복감지 → ② CVE 추출(정규식, 변형 정규화) + 로컬 DB 조회 → ③ 제품키+버전규칙 매칭 → ④ 부서별 메시지 생성·발송.
- **버전 비교기**: Windows feature update(23H2)·Office 연도·Chrome semver·Acrobat(DC 2022) 등 제품군별 체계 해석. 비교 불가 시 보수적 '후보' 처리.
- **CVE 피드 ingest**: NVD JSON / CSV / 내부 JSON 자동판별 + upsert + advisory_cve 재평가.
- **두 서버측 게이트(프론트 검증에 의존하지 않음)**: ⓐ 미등록 CVE 1건이라도 있으면 매칭 차단(409), ⓑ 발송은 명시 확인 + 멱등성 키.
- **감사 로그**: 업로드·피드적용·오탐제외·발송 등 모든 변경에 actor·시각·IP 기록.

### 3.2 발송 후(post-send) 루프 — 추가 요구 ★
업무가 끝나려면 발송 후가 채워져야 한다는 요구로 신규 구현:
| 기능 | 내용 |
|---|---|
| 조치 회신 루프 | 부서 회신 완료/진행중/**불가**(+사유) + 증빙 파일 + 권고문별 **조치율** 산출 |
| 조치결과 보고서 | **Excel(.xlsx)** + **한글 HTML→인쇄 PDF** 자동 생성(국정원/국토부 제출용) |
| SLA/D-day 트래킹 | 권고문별 D-day, 임박/기한초과, 대시보드 요약 |
| 미회신 리마인드 | 기한 임박+미회신 부서 산출 및 리마인드 발송 |
| 그룹웨어 게시판 연동 | 게시 + 댓글 회신 **웹훅 ack 동기화** (어댑터) |
| CVE 수동 추가/삭제 | 정규식 누락 보정, 게이트 재평가 |
| 오탐 제외 기억 | (자산,제품군) 조합 저장 → 다음 권고문 매칭에서 **자동 제외 제안** |

### 3.3 견고화 — 현실 데이터 대응
- **엑셀 병합/다단 헤더**: 한국 자산대장의 제목행·세로병합(부서)·2층 헤더 문제 해결. **원본 파일 불변**(메모리 grid에만 병합 forward-fill), 헤더행 자동감지 + 사용자 지정, `header_rows` 다단 결합("제품"+"OS"→"제품 / OS").
- **오프라인 번들**: React·ReactDOM·Pretendard 폰트를 `web/vendor/` 로컬화 → 외부 CDN 요청 **0건**(폐쇄망 필수). support.js 미수정(로컬 React 선로드로 CDN fetch 우회).

### 3.4 프론트엔드 연동
- **UI 템플릿 100% 유지**, 로직 클래스의 데이터 출처만 실 API로 교체(`componentDidMount` 로드 → `setState`).
- 사용자 액션을 서버에 반영: 피드 적용·매칭 진입·오탐 제외·발송·조치회신·증빙·CVE보정·게시판·자산 가져오기.
- 신규 화면 요소: CVE 수동추가 입력, 게시판 버튼, 조치율/SLA/보고서/리마인드 헤더 + 회신·증빙 버튼, 권고문 선택 드롭다운, 오탐 제외제안 배지, **자산 가져오기 실 업로드+헤더행 선택+컬럼 매핑**.

## 4. 주요 설계 결정 (왜 이렇게 했나)

| 결정 | 이유 |
|---|---|
| **SQLite 기본 + Postgres 호환** | 폐쇄망 무설치 즉시 실행. SQLAlchemy로 운영 전환 시 URL만 교체. |
| **외부 연동은 어댑터로 격리** | 메신저·메일·그룹웨어 실 규격이 §9 미확정 → 인터페이스 뒤로 격리, 미연동 시 로컬 outbox 적재(흐름 검증 가능, 외부호출 0). |
| **게이트를 서버에서 강제** | 평가 대응 신뢰성 — 프론트 우회로 미등록 CVE 매칭/중복 발송이 일어나면 안 됨. |
| **프론트는 템플릿 유지+로직만 교체** | 검증된 UX/디자인을 보존하면서 실데이터화. 표현 계산 코드 재사용. |
| **엑셀 원본 불변 + 병합 forward-fill** | 정부 자산대장은 제목행·세로병합이 흔함. 좌상단 값을 grid에만 채워 행 누락 방지. |
| **오프라인: React 로컬 선로드** | `support.js`는 GENERATED(수정 금지) → 로컬 React를 먼저 로드해 CDN fetch를 건너뛰게 함. |
| **오탐 배지: 클라이언트 표시에 서버 플래그 오버레이** | Step3 전면 서버화(고위험) 대신, 기존 표시에 `suggested_exclude`·제외상태를 덧입혀 저위험으로 동일 기능 달성. |

## 5. 검증 결과

### 5.1 자동 — 스모크 55건 전부 통과 ([smoke_test.py](../smoke_test.py))
게이트 차단(409) → 피드 적용→해제 → 매칭 → 버전비교기(Chrome<124 매칭 / Win10 제외) → 오탐제외 → 발송 → 멱등 재발송 → PDF 업로드·중복감지 → 조치회신(완료/진행/**불가 사유필수**) → 증빙 → 조치율 → 보고서(xlsx·html) → 리마인드 → CVE 추가/삭제(게이트 재평가) → 오탐 기억·재매칭 제안 → 게시판·웹훅 → 대시보드 SLA → **엑셀 제목행·세로병합(누락 0)·2층 헤더 결합**.

### 5.2 수동 — 브라우저(실 UI) 검증

## 9. 발송이력 · 조치관리 재설계 (마스터-디테일 콘솔)
드롭다운 한 줄로 보던 발송이력을 **권고문별/부서별** 리스트로 재설계. 새 독립 페이지 `/admin/history`
(정적 `web/history.html`, 레이아웃 A — 좌 리스트 / 우 상세). 기존 관리자 SPA(DC 런타임)는 무수정, 링크만.
- **조치현황 시각화**: 권고문별 부서 ack 분포(완료/진행중/불가/미회신) 누적 막대 + 조치율/회신율. `GET /history/advisories` 롤업.
- **게시판 ↔ 발송이력 동기화**: 부서 댓글 회신(ack)이 발송이력 ack 로 반영된 결과를 그대로 노출. 상세에서 관리자가 ack 직접 변경(`PATCH /notifications/{id}/ack`).
- **발송 문구 프리셋(추가/삭제)**: `MessageTemplate` 모델 + `GET/POST/DELETE /message-templates`. 미회신·진행중 부서 재발송 시 본문 템플릿({제목}{문서번호}{기한}{부서} 치환) → `POST /advisories/{id}/remind {message}`.
- **댓글 증빙 첨부**: `advisory_comment.evidence_*` 컬럼 + `POST /board/comments/{id}/evidence`(무인증). 조치상태 회신 댓글이면 해당 부서 발송이력 ack 증빙으로 자동 동기화. 열람: `GET /board/comments/{id}/evidence`, `GET /notifications/{id}/evidence`.
- **정렬/필터/검색**: 기한임박·미완료많은순·심각도 정렬, 상태 필터, 권고문·부서·담당자·문서번호 자유 검색(화면 계산).
- 테스트 `tests/test_history.py` 3건(롤업·프리셋 CRUD·댓글증빙 동기화), 전체 29건 통과.
